# PathForge TD Balance Notes

> 文档状态：2026-06-26 新增  
> 用途：记录每轮平衡调整后的验证结果，避免价格、伤害、波次和地图改动只凭感觉推进。

## 1. 记录规则

每次调整塔、敌人、波次、关卡地图或经济奖励后，至少补一条记录。记录应包含：

- 关卡 ID。
- 波次范围。
- 使用塔组合。
- 失败/通关结果。
- 漏怪类型。
- 金币剩余。
- 需要调整的数值。

自动化验证以 `GameFlow.test.ts` 为基线；手测记录用于补充 UI 可读性、操作误触、路线预览和主观难度。

## 3. 已验证设计点

| 设计点 | 当前证据 |
| --- | --- |
| 8 种塔都有实际建造价值 | `GameFlow.test.ts` 要求全部配置关卡仿真过程中实际建造到全部 `TOWER_CONFIGS` |
| 13 种敌人均被配置覆盖 | `config.test.ts` 校验波次敌人 ID 和敌人名录 |
| 关卡难度线性递增 | `config.test.ts` 计算敌人血量、抗性、速度、飞行、Boss 技能、波次密度和地图复杂度后的综合压力 |
| 飞行波需要对空体系 | `EnemyManager.test.ts` 验证飞行直线路径，波次中包含 flyer/ghost/skyBoss |
| 迷宫路径不能完全堵死 | `TowerManager.test.ts` 和 `Pathfinder.test.ts` 覆盖堵路拒绝与缓存失效 |
| 放塔交互不能清空 UI | `UIManager.test.ts` 和塔栏点击分层覆盖关键状态 |
