## 1. 执行摘要

### 1.1 一句话 Pitch

PathForge TD 是一款基于网页的像素风迷宫式塔防游戏：玩家通过摆放防御塔来塑造敌人的行进的迷宫路径，在有限资源与动态战局之间寻找最优解。

### 1.2 核心卖点

- **迷宫由你塑造**：不同于固定路径塔防，敌人在 A* 寻路下自动绕过防御塔，玩家每一次 placement 都会实时改变战局。
- **深度策略与爽快节奏并存**：短至 5-15 分钟的一局，兼顾休闲玩家与硬核策略玩家。
- **纯免费无广告**：无内购、无广告，专注玩法本身。
- **浏览器即开即玩**：无需安装，桌面与平板均可体验。

### 1.3 目标玩家

| 维度 | 描述 |
|------|------|
| 年龄层 | 12 岁以上 |
| 核心玩家 | 塔防爱好者、策略游戏玩家、独立游戏玩家 |
| 次要玩家 | 休闲玩家、像素风爱好者、网页小游戏玩家 |
| 玩家动机 | 策略优化、关卡挑战、收集解锁、速通/高分 |

### 1.4 成功标准

- 单局完成率（通关或坚持到第 20 波）≥ 60%。
- 平均单局时长 8-12 分钟。
- 玩家首次会话尝试 ≥ 3 局。
- 核心操作延迟 < 50ms，主流浏览器 60 FPS 稳定运行。

## 2. 游戏概览

### 2.1 游戏定义

| 项目 | 内容 |
|------|------|
| 游戏名称 | PathForge TD |
| 英文副标题 | Shape the Maze. Defend the Core. |
| 游戏类型 | 迷宫式塔防 / 策略 / 独立 |
| 视角 | 2D 俯视 |
| 美术风格 | 像素艺术（Pixel Art） |
| 平台 | Web 浏览器（桌面优先，平板兼容） |
| 商业模式 | 完全免费，无内购，无广告 |
| 年龄分级 | ESRB E10+ / PEGI 7 |
| 单局时长 | 5-15 分钟 |

### 2.2 核心体验

玩家扮演一名边境要塞的指挥官，在像素风格的战场上布置防御塔。敌人从地图边缘的入口涌入，试图抵达核心（玩家基地）。玩家不能直接在敌人路径上建造障碍物，但可以通过在可建造区域放置防御塔，引导敌人绕行更长的路线，从而在途中被火力消灭。

### 2.3 技术栈

| 层级 | 选型 |
|------|------|
| 渲染 | HTML5 Canvas 2D Context（WebGL 备用方案） |
| 脚本 | 原生 JavaScript（ES2022+）或 TypeScript |
| 构建工具 | Vite / 原生 ES Modules |
| 资源格式 | PNG 精灵图、WebP、Ogg / MP3 音频 |
| 状态持久化 | localStorage |
| 部署 | 静态托管（GitHub Pages / Vercel / Netlify） |

### 2.4 核心循环

```
准备阶段 → 建造/调整防御塔 → 开始波次 → 敌人沿当前路径移动 →
防御塔攻击 → 波次结束结算 → 获得金币 → 进入下一轮建造阶段 → ... → 通关或失败
```

## 3. 核心玩法循环

### 3.1 阶段拆解

#### 3.1.1 准备阶段（Pre-Wave）

- 波次开始前给予 3-5 秒缓冲时间。
- 屏幕显示「下一波敌人预览」，包括敌人类型、数量、总威胁值。
- 玩家可以在此期间建造、升级、出售防御塔。
- 路径实时重算并以半透明线条预览。

#### 3.1.2 建造阶段（Build Phase）

- 非波次进行期间均为建造阶段。
- 玩家从底部塔栏拖拽或点击选择防御塔，放置到可建造格子。
- 每次放置后触发路径验证：若导致所有入口到核心都被堵死，则放置失败并提示。
- 玩家可暂停游戏（不影响建造，仅停止时间流逝）。

#### 3.1.3 战斗阶段（Combat Phase）

- 敌人按波次配置从入口生成，沿当前最短路径移动。
- 防御塔自动索敌、攻击。
- 玩家可调整游戏速度：1x / 2x / 3x（3x 为可选解锁）。
- 敌人到达核心时扣除玩家生命。

#### 3.1.4 结算阶段（Wave Clear）

- 本波所有敌人被消灭或逃脱后进入结算。
- 根据剩余敌人数量、波次威胁值发放金币奖励。
- 记录本波数据（耗时、漏怪数、击杀数）。

### 3.2 胜利与失败

| 结果 | 条件 |
|------|------|
| 胜利 | 坚持到最后一波（通常为第 20-30 波）且生命 > 0 |
| 失败 | 生命归零 |
| 星级评价 | 通关后根据剩余生命、漏怪数、经济效率评定 1-3 星 |

### 3.3 关键规则

- **核心保护**：敌人抵达核心即扣除生命。
- **路径不可完全阻塞**：任何导致入口与核心完全断开的塔 placement 都会被拒绝。
- **动态路径**：敌人始终沿当前可用最短路径移动；若多条路径等长，随机选择一条。
- **建造不可中断波次**：波次进行中只能出售塔，不能新建或升级（特殊模式除外）。

## 4. 游戏模式与进度

### 4.1 战役模式（Campaign）

- 主线关卡序列，共 3 个章节 × 8 关 = 24 关。
- 每章引入 1-2 种新机制或新敌人类型。
- 关卡难度递增，后期引入多入口、多核心、地形障碍等。

| 章节 | 主题 | 解锁机制 |
|------|------|----------|
| 第一章：边境防线 | 草原/森林 | 基础塔、基础敌人 |
| 第二章：熔岩裂谷 | 火山/洞穴 | 元素伤害、地形岩浆 |
| 第三章：天空要塞 | 浮空城/云端 | 飞行敌人、传送门 |

### 4.2 无尽模式（Endless）

- 基于已通关地图进行无限波次挑战。
- 每 5 波难度指数级提升。
- 记录最高波次与最高分数，本地排行榜。

### 4.3 挑战模式（Challenges）

- 每周/每日轮换一个限制条件的特殊关卡。
- 示例：
  - 「只能使用箭塔」
  - 「开局金币减半」
  - 「敌人速度 +50%」
  - 「无出售」

### 4.4 进度系统

#### 4.4.1 玩家等级

- 通过完成关卡获得经验值（XP）。
- 升级解锁：新防御塔类型、新技能、更高难度。
- 等级上限：50 级。

#### 4.4.2 关卡解锁

- 通关前一关解锁下一关。
- 3 星通关可解锁该关的「困难模式」。

#### 4.4.3 成就系统

| 成就示例 | 条件 |
|----------|------|
| 完美防线 | 任意关卡 3 星通关 |
| 漏网之鱼 | 累计漏掉 100 个敌人 |
| 百万富翁 | 单局累计获得 10,000 金币 |
| 路径大师 | 使敌人路径长度超过 80 格 |

## 5. 防御塔系统

### 5.1 塔类型总览

| 塔名 | 定位 | 攻击方式 | 特殊效果 |
|------|------|----------|----------|
| 箭塔（Archer） | 基础单体 | 快速单体物理伤害 | 无 |
| 炮塔（Cannon） | 范围伤害 | 慢速炮弹，范围爆炸 | 溅射伤害 |
| 冰塔（Ice） | 控制 | 单体冰霜攻击 | 减速 |
| 雷塔（Lightning） | 连锁伤害 | 闪电链，可弹射 | 眩晕 |
| 毒塔（Poison） | 持续伤害 | 射出毒液 | 中毒 DOT |
| 狙击塔（Sniper） | 高伤害远程 | 超远射程单体 | 高暴击 |
| 兵营（Barracks） | 阻挡 | 生成士兵拦截 | 可阻挡敌人 |
| 增益塔（Support） | 辅助 | 无攻击 | 提升周围塔属性 |

### 5.2 基础属性

每座塔包含以下属性：

| 属性 | 说明 |
|------|------|
| 伤害（Damage） | 每次攻击造成的伤害值 |
| 射程（Range） | 以格子为单位的攻击半径 |
| 攻速（Fire Rate） | 每秒攻击次数 |
| 成本（Cost） | 建造所需金币 |
| 目标数（Targets） | 同时可攻击的敌人数量 |
|  projectile 速度 | 仅对弹道塔有效 |

### 5.3 升级系统

- 每座塔有 3 个等级（T1 / T2 / T3）。
- 达到 T3 后可选择一条专精分支（T4A / T4B），提供不同强化方向。
- 升级在塔信息面板中点击完成。

**示例：箭塔升级树**

```
T1 箭塔
  → T2 强化箭塔（伤害+、射程+）
    → T3 精锐箭塔（攻速+、暴击）
      → T4A 连弩塔（极高攻速，低单发伤害）
      → T4B 穿甲箭塔（穿透多个敌人）
```

### 5.4 放置规则

- 只能放置在标记为「可建造」的格子上。
- 不能放置在敌人出生点、核心周围保护区、路径格子上。
- 放置后必须保证至少存在一条从每个入口到核心的有效路径。
- 同一格子只能有一座塔。
- 波次进行中不能新建塔（部分挑战模式除外）。

### 5.5 目标优先级

防御塔默认按以下优先级索敌，玩家可手动切换：

| 优先级 | 说明 |
|--------|------|
| 优先（First） | 距离终点最近的敌人 |
| 殿后（Last） | 距离终点最远的敌人 |
| 强壮（Strong） | 当前生命值最高的敌人 |
| 虚弱（Weak） | 当前生命值最低的敌人 |
| 最近（Nearest） | 距离塔最近的敌人 |

### 5.6 出售与回收

- 出售塔返还其总投入的 70%。
- 波次进行中可以出售塔，但不会返还金币（或仅返还 50%）。
- 出售后该格恢复可建造状态，路径重新计算。

### 5.7 塔协同

| 协同组合 | 效果 |
|----------|------|
| 冰塔 + 炮塔 | 被减速敌人受到炮塔伤害 +25% |
| 雷塔 + 毒塔 | 中毒敌人被闪电命中时触发一次额外爆发伤害 |
| 增益塔 + 箭塔 | 范围内箭塔攻速 +20% |
| 兵营 + 冰塔 | 被士兵阻挡的敌人受到减速效果翻倍 |

## 6. 敌人与波次系统

### 6.1 敌人类型

| 敌人 | 特点 | 克制方式 |
|------|------|----------|
| 史莱姆（Slime） | 低血量、低速、数量多 | 群攻塔 |
| 狼（Wolf） | 低血量、高速 | 减速塔、快速单体 |
| 兽人（Orc） | 高血量、中速 | 高伤害塔 |
| 飞行者（Flyer） | 无视地面路径，直线飞向核心 | 对空塔 |
| 治疗者（Healer） | 为周围敌人回复生命 | 优先击杀 |
| 盾兵（Shielder） | 高护甲，减少物理伤害 | 魔法/毒素伤害 |
| 自爆怪（Bomber） | 死亡时对周围塔造成临时眩晕 | 远程击杀 |
| Boss | 极高血量、特殊技能、每 5 波出现 | 集火 + 控制 |

### 6.2 敌人属性

| 属性 | 说明 |
|------|------|
| 生命值（HP） | 被击败前可承受的伤害 |
| 速度（Speed） | 每秒移动格子数 |
| 护甲（Armor） | 减少物理伤害 |
| 魔抗（Magic Resist） | 减少元素伤害 |
| 奖励金币 | 被击杀后给予玩家的金币 |
| 威胁值 | 用于波次难度评估 |

### 6.3 波次配置

每波由一组或多组敌人组成，配置格式示例：

```json
{
  "wave": 5,
  "groups": [
    { "type": "slime", "count": 15, "interval": 0.4, "delay": 0 },
    { "type": "wolf", "count": 5, "interval": 0.6, "delay": 4 }
  ],
  "bonus": 120
}
```

### 6.4 难度曲线

- 每波敌人的生命值与数量按指数曲线增长。
- 基础公式：
  - `enemyHP = baseHP × (1.08 ^ waveNumber)`
  - `enemyCount = baseCount × (1.05 ^ waveNumber)`
- Boss 波：敌人生命值额外 ×3，奖励金币 ×5。

### 6.5 飞行敌人机制

- 飞行敌人不遵循地面路径，从入口直线飞向核心。
- 只有标记为「对空」的塔可以攻击飞行敌人。
- 飞行敌人不受迷宫布局影响，是玩家必须专门配置对空防御的理由。

### 6.6 Boss 设计

每个章节 Boss 拥有独特机制：

| Boss | 技能 |
|------|------|
| 森林巨兽 | 召唤小怪、短暂加速 |
| 熔岩领主 | 路径上留下燃烧地面，对附近塔造成持续伤害 |
| 天空暴君 | 召唤飞行增援，偶尔冲刺无视部分路径 |

## 7. 迷宫与寻路系统

### 7.1 网格系统

- 地图由固定大小的网格组成。
- 标准尺寸：**20 列 × 12 行**（大型地图 24×14）。
- 每格尺寸：**32×32 像素**。
- 逻辑坐标原点在左上角。

### 7.2 格子类型

| 类型 | 可建造 | 可通过 | 说明 |
|------|--------|--------|------|
| 可建造（Buildable） | 是 | 否 | 主要建造区域 |
| 路径（Path） | 否 | 是 | 敌人默认行走区域 |
| 障碍（Obstacle） | 否 | 否 | 装饰性地形 |
| 水（Water） | 否 | 否 | 不可通过，不可建造 |
| 入口（Spawn） | 否 | 是 | 敌人出生点 |
| 核心（Core） | 否 | 是 | 玩家基地，敌人目标 |

### 7.3 寻路算法

- 使用 **A*（A-Star）算法**。
- 每个入口到核心独立计算最短路径。
- 塔放置/移除/出售后立即触发路径重算。
- 若重算后某入口无路径，则拒绝该次放置。

### 7.4 路径验证规则

1. 每次放置塔前，临时将该格标记为不可通过。
2. 对每个入口运行 A* 到核心。
3. 若所有入口均存在路径，则允许放置。
4. 若任一入口无路径，则拒绝放置并显示红色提示。

### 7.5 路径预览

- 建造阶段，鼠标悬停在可建造格时，实时显示放置后的新路径（虚线）。
- 路径颜色根据路径长度变化：绿色（安全）、黄色（适中）、红色（过短/危险）。

### 7.6 多入口与多核心

- 后期关卡支持多个敌人入口。
- 部分关卡存在多个核心，任一核心被突破均扣除生命。
- 寻路需分别计算每个入口到最近核心的路径。

## 8. 经济系统

### 8.1 货币类型

| 货币 | 用途 | 获取方式 |
|------|------|----------|
| 金币（Gold） | 建造/升级塔 | 起始金币、击杀、波次奖励 |
| 经验值（XP） | 玩家等级、解锁内容 | 完成关卡、成就 |

> 注：因游戏完全免费，不存在任何付费货币或广告货币。

### 8.2 收入来源

| 来源 | 说明 |
|------|------|
| 起始金币 | 每关开始时给予，用于初始布局 |
| 击杀奖励 | 消灭敌人时立即获得 |
| 波次奖励 | 每波结束后根据表现发放 |
| 利息（可选） | 波次结束时，未花费金币的 10% 作为奖励 |

### 8.3 成本曲线

- 塔基础成本与升级成本遵循递增曲线。
- 示例成本表（箭塔）：

| 等级 | 建造/升级成本 | 累计投入 |
|------|---------------|----------|
| T1 | 80 | 80 |
| T2 | 120 | 200 |
| T3 | 200 | 400 |
| T4A / T4B | 350 | 750 |

### 8.4 出售返还

- 普通出售返还总投入的 70%。
- 波次中出售返还 50%。
- 返还金币向下取整。

### 8.5 经济平衡原则

- 玩家在正确布局下应能通过正常升级应对前 70% 的波次。
- 后期波次需要依赖路径优化、塔协同和专精选择。
- 不出现必须依赖随机性的经济压力。

### 8.6 初始关卡经济示例

| 关卡 | 起始金币 | 通关奖励金币 | 推荐路径长度 |
|------|----------|--------------|--------------|
| 1-1 | 400 | 150 | 18 格 |
| 1-4 | 550 | 220 | 24 格 |
| 2-1 | 700 | 280 | 28 格 |
| 3-1 | 900 | 350 | 32 格 |

## 9. 关卡设计

### 9.1 关卡结构

每个关卡包含：

- 一个或多个敌人入口。
- 一个或多个玩家核心。
- 可建造区域与不可建造地形。
- 20-30 波敌人配置。
- 可选的环境机制（陷阱、传送门、障碍物）。

### 9.2 地图尺寸标准

| 尺寸 | 列 × 行 | 用途 |
|------|---------|------|
| 小型 | 16 × 10 | 教学关、挑战关 |
| 标准 | 20 × 12 | 常规关卡 |
| 大型 | 24 × 14 | 后期关卡、无尽模式 |

### 9.3 地形类型

| 地形 | 效果 |
|------|------|
| 草地 | 无特殊效果 |
| 森林 | 可建造，敌人通过时速度 -20% |
| 岩浆 | 不可建造，敌人通过时持续掉血 |
| 水域 | 不可建造，不可通过 |
| 高地 | 射程 +1，但敌人不可通过 |
| 传送门 | 敌人进入后从另一处传送门出现 |

### 9.4 关卡难度设计原则

- 早期关卡：开阔地形，引导玩家学习迷宫构建。

## 10. 战斗与特效

### 10.1 伤害计算

基础伤害公式：

```
实际伤害 = max(1, 基础伤害 × 伤害类型倍率 - 目标护甲)
```

- 物理伤害：受护甲减免。
- 元素伤害（火/冰/雷/毒）：受魔抗减免。
- 真实伤害：无视护甲与魔抗。

### 10.2 伤害类型

| 类型 | 代表塔 | 特点 |
|------|--------|------|
| 物理 | 箭塔、炮塔、兵营 | 受护甲影响 |
| 火焰 | 火焰塔 | 可造成灼烧 DOT |
| 冰霜 | 冰塔 | 减速效果 |
| 雷电 | 雷塔 | 可弹射、可眩晕 |
| 毒素 | 毒塔 | 持续伤害、可叠加 |

### 10.3 状态效果

| 效果 | 说明 | 叠加规则 |
|------|------|----------|
| 减速（Slow） | 移动速度 -30% | 不叠加，取最高值 |
| 灼烧（Burn） | 每秒受到伤害 | 可刷新持续时间 |
| 中毒（Poison） | 每秒受到伤害 | 可叠加层数，最高 5 层 |
| 眩晕（Stun） | 无法移动 | 不叠加 |
| 破甲（Armor Shred） | 护甲 -50% | 不可叠加 |
| 冰冻（Freeze） | 完全停止 | 不叠加，受火伤后解除 |

### 10.4 投射物与命中

| 攻击方式 | 说明 |
|----------|------|
| 弹道（Projectile） | 有飞行时间，可被躲避或拦截 |
| 即时命中（Hitscan） | 立即命中，如狙击塔 |
| 范围伤害（AoE） | 命中区域造成范围伤害 |
| 持续伤害（Beam） | 持续照射，逐帧伤害 |

### 10.5 视觉反馈

- 敌人受击时闪烁白色。
- 暴击时显示橙色伤害数字。
- 状态效果在敌人头顶显示小图标。
- 范围伤害落地时显示爆炸特效。
- 核心受击时屏幕轻微震动。

## 14. 技术架构

### 14.1 整体架构

```
Game
├── Renderer (Canvas 2D)
├── StateManager
├── InputManager
├── EntityManager
│   ├── TowerManager
│   ├── EnemyManager
│   ├── ProjectileManager
│   └── EffectManager
├── WaveManager
├── Pathfinder (A*)
├── EconomyManager
├── UIManager
├── AudioManager
└── SaveManager
```

### 14.2 游戏循环

```javascript
function gameLoop(timestamp) {
  const dt = timestamp - lastTime;
  update(dt);   // 逻辑更新
  render();     // 渲染
  lastTime = timestamp;
  requestAnimationFrame(gameLoop);
}
```

- 目标帧率：60 FPS。
- 使用固定时间步长处理逻辑，渲染使用插值。
- 支持游戏速度倍率（1x / 2x / 3x）。

### 14.3 渲染管线

1. 清空画布。
2. 绘制地图底层（地形）。
3. 绘制路径与路径预览。
4. 绘制防御塔。
5. 绘制敌人。
6. 绘制投射物与特效。
7. 绘制 UI 覆盖层。

### 14.4 性能目标

| 指标 | 目标 |
|------|------|
| 帧率 | 桌面 60 FPS，平板 30 FPS 以上 |
| 同屏敌人上限 | 80 个 |
| 同屏投射物上限 | 200 个 |
| 首屏加载时间 | ≤ 3 秒 |
| 内存占用 | ≤ 200 MB |

### 14.5 路径搜索优化

- A* 使用二叉堆优先队列。
- 仅在塔状态变化时重新计算路径。
- 缓存有效路径，避免每帧重新寻路。
- 大型地图使用分块寻路或增量更新策略。

### 14.6 输入处理

- 鼠标/触摸事件统一抽象为 pointer 事件。
- 点击检测使用网格坐标映射。
- 拖拽建造支持触摸屏。

### 17.2 核心公式速查

```
实际伤害 = max(1, 基础伤害 × 类型倍率 - 护甲)
敌人生命 = 基础生命 × (1.08 ^ 波次)
敌人数量 = 基础数量 × (1.05 ^ 波次)
塔升级成本 = 基础成本 × 1.6 ^ (等级 - 1)
出售返还 = 总投入 × 0.7
```
