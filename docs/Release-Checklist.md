# PathForge TD Release Checklist

> 用途：发布前统一检查构建、测试、玩法、文档和 GitHub 状态。每次准备可试玩版本或正式版本时执行。

## 1. 版本准备

- [ ] 确认 `package.json` 版本号是否需要更新。
- [ ] 更新 `CHANGELOG.md`，记录新增、修复、测试和已知问题。
- [ ] 确认 `README.md` 的功能规模和运行方式与当前实现一致。
- [ ] 确认 `docs/README.md` 文档索引没有失效链接。
- [ ] 若调整平衡，追加 `docs/Balance-Notes.md`。

## 2. 自动化验证

```bash
npm run test -- --run
npm run build
```

验收：

- [ ] Vitest 全部通过。
- [ ] TypeScript 编译通过。
- [ ] Vite 生产构建通过。
- [ ] 构建产物在 `dist/` 下生成。

## 3. 核心手测

建议至少覆盖以下路径：

| 场景 | 检查点 |
| --- | --- |
| 主菜单 | 战役、无尽、设置、统计页面可进入和返回 |
| 关卡选择 | 已解锁关卡可进入，锁定关卡状态清晰 |
| 1-1 | 新玩家教学节奏，塔说明、范围预览、路径塑形清晰 |
| 1-2 | 熔岩/森林地形显示和规则可理解 |
| 1-3 | 双入口、多核心、飞行敌人路径清晰 |
| 8-3 | 高压终局波次不会卡死或明显掉帧 |
| 胜利结算 | 星级、继续下一关、回菜单流程正常 |
| 失败结算 | 重试、回菜单流程正常 |
| 设置页 | 音量、画质、路径预览、粒子开关显示不重叠 |
| 成就页/统计页 | 100 个成就和统计项显示稳定 |

## 4. 放塔与 UI 回归

- [ ] 选中任意塔后，鼠标移动到地图显示作用范围。
- [ ] 金币不足时，塔卡禁用或提示清晰。
- [ ] 不可建造格子放塔失败后，选中塔和说明面板不消失。
- [ ] 堵路放塔失败后，HUD、塔栏和波次预览不消失。
- [ ] 连续成功/失败放塔 20 次，界面层级稳定。
- [ ] 点击已有塔后，升级、出售、目标优先级操作正常。

## 5. 寻路与地图

- [ ] 地图格子运行中不移动。
- [ ] 地面路径预览只改变路径线，不改变地图布局。
- [ ] 任一入口到核心始终至少有一条可行路径。
- [ ] 飞行敌人路径和地面敌人路径视觉区分明确。
- [ ] 多入口关卡中，任一入口都不会因合法放塔被完全堵死。

## 6. 存档与兼容

- [ ] 新存档可以正常开始战役和无尽模式。
- [ ] 旧存档字段缺失时，`SaveManager` 迁移后不报错。
- [ ] 通关后关卡解锁、星级、统计和成就写入正常。
- [ ] 设置项刷新页面后保持。

## 7. 发布步骤

1. 确认工作区干净：

   ```bash
   git status --short
   ```

2. 创建发布提交：

   ```bash
   git add -A
   git commit -m "chore: release vX.Y.Z"
   ```

3. 推送主分支：

   ```bash
   git push origin main
   ```

4. 可选：创建 Git tag：

   ```bash
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```

5. 在 GitHub Release 中粘贴 `CHANGELOG.md` 对应版本内容。

## 8. 发布后检查

- [ ] GitHub 仓库首页 README 正常显示。
- [ ] Release 页面版本号、说明和链接正确。
- [ ] 线上或预览环境可以正常加载游戏。
- [ ] 控制台没有启动即报错。
- [ ] 若发现阻断问题，立即记录到 `CHANGELOG.md` 的已知问题或创建修复任务。
