# PathForge TD Testing Guide

本文档定义当前仓库的自动化测试、手测和回归标准。

## 1. 测试目标

测试重点不是只证明代码能运行，而是保护塔防游戏最容易回归的行为：

- 放塔不能堵死所有路径。
- 放塔成功或失败后 UI 状态必须稳定。
- 关卡、波次、塔、敌人和成就配置互相一致。
- 地面敌人按路径移动，飞行敌人不依赖地面路径。
- 胜负、下一关、存档和成就流程不破坏旧进度。

## 2. 常用命令

安装依赖：

```bash
npm install
```

运行全部测试：

```bash
npm run test -- --run
```

监听测试：

```bash
npm run test
```

构建检查：

```bash
npm run build
```

本地开发：

```bash
npm run dev
```

本地预览生产构建：

```bash
npm run preview
```

## 3. 当前自动化覆盖

| 测试文件 | 关注点 |
| --- | --- |
| `src/config/config.test.ts` | 塔、敌人、波次、成就配置完整性和引用关系 |
| `src/config/levels.test.ts` | 24 个关卡基础合法性和流程仿真 |
| `src/systems/Pathfinder.test.ts` | A* 路径、放塔校验、缓存失效 |
| `src/systems/TowerManager.test.ts` | 放塔、升级、出售、路径阻断防护 |
| `src/systems/EnemyManager.test.ts` | 敌人生成、移动、到达核心和死亡 |
| `src/systems/GameFlow.test.ts` | 游戏流程、波次和胜负相关行为 |
| `src/entities/Tower.test.ts` | 塔攻击、升级和目标选择 |
| `src/entities/Projectile.test.ts` | 投射物移动、命中和范围效果 |
| `src/ui/UIManager.test.ts` | UI 状态保持、塔说明和交互边界 |

新增或修改共享行为时，应优先补对应层级的测试，而不是只依赖手测。

## 4. 配置测试标准

配置改动至少验证：

- 所有塔都有 `role / description / special / usage`。
- 所有波次敌人 ID 都存在于 `ENEMY_CONFIGS`。
- 所有关卡尺寸和 `grid` 行列一致。
- 所有关卡都有至少一个 `spawn` 和一个 `core`。
- 所有关卡初始状态下每个入口至少能到达核心。
- 所有成就条件可由当前存档和运行时上下文判断。

## 5. 放塔与寻路测试标准

必须覆盖：

- 可建造格能正常放塔并扣金币。
- 不可建造格、已占用格、金币不足和堵死路径都放塔失败。
- 放塔失败不扣金币、不写入 `towerId`。
- 放塔失败后 `selectedTowerId` 保留。
- 成功放塔和出售后 `Pathfinder.invalidate()` 生效。
- 多入口关卡中任意入口都不能被堵死。
- 地形权重影响路线，但不改变格子坐标。

## 6. UI 回归手测

每次 UI、菜单、CSS 或 Canvas 渲染改动后，至少手测：

1. 主菜单：按钮对齐、标题清晰、无文本溢出。
2. 选关页：24 个关卡可见，星级和关卡名不挤压。
3. 设置页：滑块、开关、画质选项和重置按钮不重叠。
4. 战斗 HUD：生命、金币、波次、速度和暂停状态可读。
5. 底部塔栏：8 张塔卡都能展示价格、定位和状态。
6. 塔说明：点击任意塔卡后能看到职责、特殊效果和推荐用法。
7. 范围预览：选中塔并悬停地图时显示范围圈和合法性。
8. 放塔失败：UI 不消失，塔仍保持选中。
9. 暂停菜单：恢复、重开、选关、主菜单可点击。
10. 胜利结算：出现下一关入口；最后一关不显示无效下一关。
11. 失败结算：重开和返回入口可用。
12. 统计/成就页：卡片排布不重叠。

## 7. 关卡手测矩阵

| 关卡类型 | 建议样本 | 必测点 |
| --- | --- | --- |
| 教学基础 | `1-1` | 初始经济、基础路径、普通敌人 |
| 地形关 | `1-2`、`2-1` | 熔岩、森林、水和障碍是否服务路线 |
| 飞行关 | `1-3` | 飞行路线、防空塔、路径预览 |
| 双入口 | `2-2`、高级 stage 2 | 每个入口都不能被堵死 |
| 多核心 | `2-3`、高级 stage 3 | 多核心寻路和结算稳定 |
| 高压后期 | `7-3`、`8-3` | Boss、飞行、经济和性能 |

## 8. 浏览器检查

建议至少覆盖：

- Chromium 或 Chrome。
- 桌面常见窗口：1280x720、1440x900。
- 窄窗口：宽度接近 960px 时，菜单和塔栏仍可读。

检查内容：

- 控制台无红色错误。
- Canvas 非空，地图、塔、敌人、路径和 UI 都正常绘制。
- 鼠标坐标与格子位置一致，缩放后点击不偏移。
- 设置改动刷新页面后仍生效。
- 存档损坏时不会阻止游戏启动。

## 9. 发布前最低验证

发布前必须完成：

```bash
npm run test -- --run
npm run build
```

并手测：

- `1-1` 从主菜单进入、放塔、开波、通关、进入下一关。
- 一个双入口关卡的路径阻断失败场景。
- 一个飞行波次。
- 设置页修改和重置。
- 统计/成就页打开。
