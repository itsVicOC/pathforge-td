# PathForge TD UI & Art Guide

本文档定义当前项目的 UI、Canvas 美术和视觉验收标准。它用于避免界面回退到不可读、错位或塔/敌外观难以区分的状态。

## 1. 视觉目标

PathForge TD 的界面应像“战术奇幻塔防工具台”：

- 信息密度高，但层级清楚。
- 塔、敌人、地形和路径一眼可区分。
- UI 服务决策，不做遮挡玩法的装饰。
- 菜单、设置、结算和战斗内 HUD 使用统一材质、间距、圆角和按钮语言。

## 2. 信息层级

战斗界面优先级：

1. 地图和路径。
2. 敌人、塔、投射物、特效。
3. 当前悬停格和塔作用范围。
4. HUD：生命、金币、波次、速度。
5. 底部建造栏。
6. 塔说明面板和选中塔面板。
7. 下一波预览。
8. 暂停、胜利、失败等模态层。

放塔交互不能清空第 4-7 层。

## 3. 塔卡规范

每张塔卡必须展示：

- 名称。
- 价格。
- 定位标签。
- 简短职责。
- 是否买得起。
- 是否当前选中。

选中塔后，说明面板必须展示：

- 定位。
- 目标类型。
- 伤害类型。
- 范围、伤害、攻速。
- 特殊效果。
- 推荐用法。

不要只用颜色区分塔；必须有图形语言差异，例如塔身、炮口、尖顶、线圈、瓶罐、瞄准镜、旗帜或兵营结构。

## 4. 放置预览规范

玩家选中塔并移动到地图时：

- 在悬停格绘制放置预览。
- 绘制以塔为中心的作用范围。
- 合法位置和非法位置使用明显不同的颜色和描边。
- 鼠标位于 UI 上时不显示地图悬停格。
- 放置失败后仍保留当前选中塔和说明面板。

范围圈必须和实际 `Tower.getRange()` 或配置范围一致，不能为了视觉夸大。

## 5. HUD 规范

HUD 必须保持：

- 生命、金币、波次、速度总是可见。
- 数字变化不导致布局跳动。
- 暂停和速度状态有明确反馈。
- HUD 不遮挡入口、核心或关键路径。

如果新增 HUD 信息，需要优先考虑是否会压缩地图可读性。

## 6. 菜单和设置页规范

菜单页要求：

- 主菜单按钮垂直或网格排列清晰。
- 选关页可一次浏览大量关卡，不让关卡名和星级重叠。
- 设置页控件按行对齐，标签、当前值和输入控件有稳定宽度。
- 统计和成就页可滚动，但内容不应被底部按钮遮挡。
- 暂停和结算面板要突出主要行动：继续、重开、下一关、返回。

设置页特别要求：

- 滑块行高一致。
- 开关状态文本不挤压滑块。
- 小窗口下优先换行，不允许控件互相覆盖。

## 7. 地图视觉规范

地图格子必须稳定：

- 运行中格子坐标不移动。
- 路径预览只改变路径线，不改变地图布局。
- 地形材质可以有纹理，但不能让格子边界不可读。
- 入口和核心必须是第一眼可识别的目标。

地形建议：

| 类型 | 视觉关键词 | 玩法含义 |
| --- | --- | --- |
| `path` | 压实道路、清晰主路 | 低成本地面通路 |
| `buildable` | 可部署地面 | 可放塔，寻路成本较高 |
| `forest` | 树丛/草地 | 可放塔，移动减速 |
| `lava` | 裂隙/熔流 | 可走但危险，成本高 |
| `water` | 水面 | 不可通行 |
| `obstacle` | 岩石/废墟 | 不可通行 |
| `spawn` | 入口门/旗标 | 敌人来源 |
| `core` | 核心水晶/基地 | 防守目标 |

## 8. 敌人视觉规范

敌人需要同时通过颜色和轮廓区分：

- 高速敌人：更尖锐或带速度线。
- 盾兵：盾牌或厚重外壳。
- 治疗者：支援标记或光环。
- 自爆怪：警示色和爆炸暗示。
- 飞行敌人：翅膀、阴影或悬浮感。
- Boss：体型更大、轮廓复杂、血条醒目。

血条必须在不同背景上可读，Boss 或精英单位可以有更明显的状态指示。

## 9. 投射物和特效规范

投射物要帮助玩家判断塔的职责：

- 箭塔：细长弹体，速度感强。
- 炮塔：较大弹体，命中有范围反馈。
- 冰塔：冷色轨迹和减速反馈。
- 雷塔：即时或链式光束。
- 毒塔：绿色弹体和持续伤害反馈。
- 狙击塔：细亮线或瞬时命中。
- 兵营：重击、震荡或眩晕反馈。

特效不能遮挡关键 UI，粒子开关关闭时仍要保留必要命中反馈。

## 10. 响应式标准

当前游戏以桌面浏览器体验为主，但应保证常见窗口尺寸不崩坏：

- 不使用随视口宽度无限变化的字体大小。
- 文本不能溢出按钮、卡片或面板。
- 底部塔栏在较窄窗口下可换行或压缩间距。
- 选关、统计、成就等长内容必须可滚动。
- Canvas 点击坐标和视觉位置一致。

## 11. 禁止回退

以下情况视为 UI/美术回归：

- 塔仅由简单纯色圆形或方块表示，无法区分职责。
- 放塔导致 HUD、塔栏或说明面板消失。
- 设置页控件错位或互相覆盖。
- 关卡结算后没有明确下一步。
- 地图格子在运行中移动。
- 大面积无意义空地让路线塑形失去意义。
- 路径线、范围圈或飞行路线混在一起无法辨认。

## 12. 视觉改动验收

视觉改动完成后至少检查：

- 主菜单。
- 选关页。
- 设置页。
- 统计/成就页。
- `1-1` 战斗界面。
- 任意双入口关卡。
- 任意飞行波次。
- 胜利结算和下一关。
- 失败结算。

推荐同时运行：

```bash
npm run test -- --run
npm run build
```
